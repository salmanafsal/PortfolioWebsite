STAR Reflection – Pokémon Squad Feature

Situation: We needed to enhance a FastAPI-based Pokémon app so that users could build a squad of Pokémon using server-rendered HTML.

  • Retrieve the full list of 151 Pokémon via a FastAPI endpoint.
  • Allow users to add Pokémon to their squad up to a maximum of six.
  • Display the current squad on the same page.
  • Show an error message if the user attempts to select more than six Pokémon.
  • (Bonus) Enable a “Battle” button once at least two Pokémon are selected.

Task: My role was to implement the squad-building logic using FastAPI backend and HTML templates—without using React or client-side frameworks.

  • Designed endpoints to handle fetching Pokémon and updating the squad selection on the backend.
  • Used server-side HTML templating to render the list of Pokémon and the squad dynamically.
  • Rendered error messages and update the “Battle” button state based on squad size.
  • Showed an error message if the user attempts to select more than six Pokémon.
  • (Bonus) Enabled a “Battle” button once at least two Pokémon are selected.

Action:

  • API & Templating: Created FastAPI GET / endpoint to load all Pokémon into an HTML template using Jinja2. A POST /add endpoint handled form submissions to add Pokémon to the squad.
  • Server-Side Logic: Employed session or simple in-memory state to track the user’s squad. The add handler verified uniqueness and enforced the six-Pokémon limit. If the squad was already full or contained the selected Pokémon, the page was re-rendered with appropriate error text.
  • Conditional Rendering : In the Jinja2 template, I looped through the squad state to render selected Pokémon cards. The “Battle” button was rendered disabled or enabled based on len(squad) >= 2.
  • Error Feedback : Embedded error message alerts in the template when the POST handler detected rule violations.

Result:

  • Functional Feature: Users can now build a squad by adding up to six unique Pokémon, with real-time feedback and no need for client-side frameworks.
  • User Experience: The app cleanly renders squad additions, changes, and error messages on each request cycle—all handled server‑side.
  • Future‑Ready: The “Battle” button’s enabled/disabled state naturally supports adding Pokémon to a server‑side battle queue using FastAPI in the future.
  • Skill Growth: This setup deepened my expertise in adding robust FastAPI routes, server‑side templating with Jinja2, and managing form‑driven UI logic without frontend frameworks.
